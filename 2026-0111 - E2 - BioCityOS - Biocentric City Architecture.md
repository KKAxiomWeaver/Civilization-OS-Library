## 📝 Abstract

**Biocentric City OS（BioCityOS）** 定義一種「以生態為中心、以文明為客體」的城市設計 OS。  
在 Earth 2.0 的順勢文明線下，城市不再是 **對抗自然的堡壘**，而是嵌入行星–生態–文明壓力流中的一個「柔性節點」。

BioCityOS 核心假設：

> **城市若無視生態與行星調節機制，就等同於在壓力場中心蓋一個放大器。**  
> **相反，若城市被設計成壓力緩衝與共生節點，它會成為 PlanetaryRegOS 的合作方，而非對手。**

本白皮提供：

- 一套 biocentric 城市架構（緩衝帶、低壓區、墊層、物種共棲區）  
- 人類通行 / 活動 / 工作節奏如何與「棲地節奏」錯峰運作  
- 舊式城市（AnthroCity） vs BioCity 的壓力曲線差異  
- 從大安森林公園 / 寶可夢 2.0 的案例，抽象出可重用的城市 OS 模型

## 01 — Problem Statement

### 1.1 現行城市設計的核心錯誤

Legacy 城市（AnthroCity）大多遵循：

- 人的動線優先  
- 車的流量優先  
- 土地價值優先  
- 24/7 燈光＋噪音  
- 草地、樹木、生物被當「綠化裝飾」  

結果：

- 生態被排擠到邊界（甚至城市外）  
- 城市成為 **高密度壓力集中點**  
- 噪音 / 光害 / 密度 / 熱島效應 → 形成常態性高壓場  
- 城市本身變成 StressDynOS 的壓力尖峰區  

### 1.2 導致什麼？

- 高文明病率（敘事層面：焦慮、過勞、自殺率上升）  
- 異常生態反應（動物闖入、棲地崩壞）  
- 高種族值 / 關鍵物種必須頻繁調節  
- 城市變成 PlanetaryRegOS 的「問題點」，而非合作節點  

### 1.3 BioCityOS 要解決的根本問題

- 如何把城市從「壓力放大器」變成「壓力緩衝器」  
- 如何讓城市結構本身就具備「順勢」的特性  
- 如何讓城市可以被 PlanetaryRegOS + MultiSpeciesOS 視為合作節點

## 02 — Concept Model

### 2.1 Biocentric City（BioCity）定義

> **BioCity = 一座預設自己是「棲地的一部分」而非「棲地之上的異物」的城市。**

特徵：

- 城市視自己為 **Habitat Layer** 的一層，不是替代品  
- 人類與其他物種共享空間，但不強求「所有空間都能被人使用」  
- 城市內存在明確的：  
  - 人類專用路徑  
  - 物種專用路徑  
  - 共用與緩衝區  

### 2.2 BioCityOS 的三大原則

1. **圍不是城，而是壓力場**  
   - 城市邊界不以牆 / 道路定義，而是以「壓力梯度」定義。  

2. **不滿格運作原則**  
   - 城市不追求 100% 容量與利用率，  
     預留壓力吸收空間（如身體預留修復能量）。  

3. **棲地優先原則**  
   - 某些區域，地位高於「人類擁有權」，稱為 **生態優先區**。  
   - 城市僅能在此區域以「訪客」方式存在。

## 03 — Mechanics（How It Works）

### 3.1 城市壓力分層

BioCityOS 將城市區分為四個圈層（可抽象應用）：

1. 🟢 **Core Flow Zone（核心流動區）**  
   - 人口密度中等  
   - 壓力可控  
   - 大量「綠島」與緩衝點  
   - 生活節奏與白天 / 黑夜一致  

2. 🌿 **Green Buffer Belt（綠色緩衝帶）**  
   - 城市與棲地之間的過渡  
   - 弱勢與中階物種主要活動區  
   - 降低噪音 / 光害 / 交通衝擊  

3. 🌲 **Wild Mesh Zone（自然網絡區）**  
   - 大型棲地、森林、水體  
   - 高種族值 / 關鍵物種偶爾巡域  
   - 人類僅作短暫活動，不做固定建設  

4. 🌫 **HighReg Zone（行星調節區）**  
   - 如山脈、海溝、極地  
   - 對應 PlanetaryRegOS  
   - 城市完全不得侵入  

### 3.2 城市節奏與生態節奏的錯峰

BioCityOS 定義：

- 人類高活動時段  
- 寶可夢 / 動物高活動時段  
- 行星調節時段（如氣候翻轉、巡域時刻）  

透過錯峰設計：

- 人不會在最高壓力時段被曝露在自然反應中  
- 動物 / 寶可夢不會被人類活動干擾自身調節

## 04 — Architecture

### 4.1 Layered City OS

- **Human Mobility Layer**  
  - 步行、低速交通為主軸  
  - 車只在特定走廊存在  

- **Bio Mobility Layer**  
  - 樹冠走廊（鳥 / 昆蟲）  
  - 綠帶走廊（地面物種）  
  - 地下 / 水道走廊（小型物種）  

- **Pressure Layer**  
  - Noise map  
  - Light map  
  - Crowd density map  

- **Interface Layer**  
  - 人類與生態的共享節點（公園、共棲廣場、低壓街區）  

### 4.2 接入其他 OS 的方式

- FlowCivOS：  
  - BioCityOS 是 FlowCivOS 在城市尺度的具體實作。  

- StressDynOS：  
  - BioCityOS 為壓力提供「日常釋放管道」。  

- MultiSpeciesOS：  
  - 城市設計反向讀取物種行為（棲地、遷徙、壓力反應）。  

- PlanetaryRegOS：  
  - 不在 PlanetaryReg 的 high-tension 線上建高密度城市。

## 05 — Use Cases

1. **大安森林公園實驗（示意）**  
   - 分出人類高活動區 / 寶可夢休息區 / 緩衝草帶  
   - 高種族值不再靠近城市邊界，壓力顯著下降。  

2. **夜間減光政策**  
   - 降低光害，讓 flying / ghost 系寶可夢自然活動  
   - 同時改善人類睡眠節律。  

3. **沿海 BioCity 帶**  
   - 將部分港區改為「自然緩衝港」  
   - 減低海洋壓力波反彈進城市。  

4. **震帶城市設計**  
   - 與 PlanetaryRegOS 聯動：  
     - 在高板塊張力區避免高密度建設  
     - 以 BioCity 模式做低密度、彈性城市。

## 06 — Risks & Limitations

- **政治現實**：  
  - 既得利益與開發模式難以馬上轉換。  

- **資料不足**：  
  - 精確畫出壓力場 / 物種路徑，需要長期觀測。  

- **都市既有結構難以大幅更改**：  
  - BioCityOS 初期較適合新區域或重建區。  

- **被當成裝飾性綠化**：  
  - 若只做表面植栽，而未改變節奏與流向，本 OS 失效。

## 07 — Comparative Analysis

### vs. 傳統「人類中心城市」

- 傳統：  
  - 交通導向，車優先，綠地點狀  
- BioCityOS：  
  - 壓力導向、棲地導向，多物種導向  

### vs. 單純「綠色城市 / Garden City」

- Garden City：  
  - 以美學與綠量為主  
- BioCityOS：  
  - 以「壓力流」與「物種路徑」為核心架構

## 08 — Implementation Path

### Stage I — Existing City Patch

- 製作 Noise / Light / Density Maps  
- 選幾處做「低壓節點」  
  - 夜間減光  
  - 禁車區域  
  - 生態友善路徑  

### Stage II — 新開發區 Biocentric 設計

- 將四層圈（Core Flow / Buffer / Mesh / HighReg）  
  納入土地規劃。  

### Stage III — BioCity Network

- 多座 BioCity 串聯成 **Biocentric Mesh**，  
  成為行星與文明的「合作網格」。
